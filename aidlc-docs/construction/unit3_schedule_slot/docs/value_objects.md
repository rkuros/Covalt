# Unit 3: 値オブジェクト一覧

## 設計判断の根拠

- 時刻は HH:mm 形式 + 日付は別フィールド、JST 固定（Q4）
- 所要時間: 最小 15 分、最大 1440 分（Q5）
- スロット status は available / booked の 2 値（Q2）

## 7. TimeOfDay

1 日の中の時刻を表す値オブジェクト。タイムゾーンは JST 固定のため、時刻のみを保持する。

### 属性

| フィールド | 型 | 説明 |
|-----------|------|------|
| hour | integer | 時（0〜23） |
| minute | integer | 分（0〜59） |

### バリデーションルール

- hour: 0 以上 23 以下
- minute: 0 以上 59 以下

### 表示形式

- HH:mm 形式（例: "09:00", "17:30"）
- PACT 契約の `startTime` / `endTime` フィールドに対応

### 等価性

- hour と minute がともに同一であれば等価

### 比較

- 大小比較が可能（isBefore / isAfter）
- hour を優先し、同一の場合は minute で比較

## 8. TimeRange

開始時刻と終了時刻の組で時間帯を表す値オブジェクト。BusinessHour の営業時間帯やスロットの時間枠を表現する。

### 属性

| フィールド | 型 | 説明 |
|-----------|------|------|
| startTime | TimeOfDay | 開始時刻 |
| endTime | TimeOfDay | 終了時刻 |

### バリデーションルール

- startTime は endTime より前であること（startTime < endTime）
- startTime と endTime はいずれも有効な TimeOfDay であること

### ビジネスメソッド

- **overlaps(other: TimeRange): boolean** -- 他の TimeRange と時間帯が重複するかを判定する。スロット重複チェック（Q10: 重複不可）に使用
- **contains(time: TimeOfDay): boolean** -- 指定時刻がこの時間帯に含まれるかを判定する
- **durationInMinutes(): integer** -- 時間帯の長さを分単位で返す

### 等価性

- startTime と endTime がともに同一であれば等価

## 10. SlotStatus

スロットの状態を表す値オブジェクト。

### 属性

| フィールド | 型 | 説明 |
|-----------|------|------|
| value | enum | AVAILABLE / BOOKED |

### バリデーションルール

- AVAILABLE または BOOKED のいずれかであること

### PACT 対応

- AVAILABLE は PACT 上 `"available"` にシリアライズ
- BOOKED は PACT 上 `"booked"` にシリアライズ

### 等価性

- value が同一であれば等価

---

## 11. Duration

所要時間を表す値オブジェクト。スロットの長さを分単位で表現する。

### 属性

| フィールド | 型 | 説明 |
|-----------|------|------|
| minutes | integer | 所要時間（分） |

### バリデーションルール

- 最小値: 15 分（Q5）
- 最大値: 1440 分（24 時間）（Q5）
- 整数であること

### PACT 対応

- PACT 上 `durationMinutes` フィールドに対応（integer 型）

### 等価性

- minutes が同一であれば等価

---
