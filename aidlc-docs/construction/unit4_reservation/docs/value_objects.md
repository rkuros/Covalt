# Unit 4 予約管理 - 値オブジェクト定義

## 概要

Unit 4（予約管理）の境界づけられたコンテキストに属する値オブジェクトを定義する。
値オブジェクトはアイデンティティを持たず、属性の等価性によって同一性を判定するドメインオブジェクトである。
すべての値オブジェクトはイミュータブル（不変）とする。

---

## 1. ReservationId（予約ID）

予約を一意に識別するための値オブジェクト。

| 項目 | 内容 |
|------|------|
| 内部表現 | UUID v4 形式の文字列 |
| 生成ルール | 予約作成時にシステムが自動生成 |
| 制約 | RFC 4122 UUID v4 準拠。null 不可 |
| 等価性 | UUID 文字列の一致で判定 |
| 根拠 | 分散環境で衝突なし、推測不可、標準ライブラリで生成可能（Q19 確定） |

備考: PACT のサンプル値では `rsv-001` のようなプレフィックス付き形式を使用しているが、実装では UUID v4 を採用する。PACT の値はテスト用のダミーである。

---

## 2. ReservationStatus（予約ステータス）

予約のライフサイクルにおける状態を表す列挙型の値オブジェクト。

| 値 | 表示名 | 説明 |
|----|--------|------|
| `confirmed` | 確定 | 予約が作成され有効な状態。変更後もこのステータスを維持する |
| `cancelled` | キャンセル済み | 顧客またはオーナーにより取り消された状態（終端） |
| `completed` | 完了 | サービス提供が完了した状態（終端） |

設計判断:
- 3値とする。`modified` は独立ステータスとしない（Q17 確定）
- 変更操作では `confirmed` のまま日時のみ更新し、変更の記録は ReservationHistory に委ねる
- `cancelled` と `completed` は終端状態であり、そこからの遷移は不可

### 許可される状態遷移

| 遷移元 | 遷移先 | 備考 |
|--------|--------|------|
| confirmed | cancelled | キャンセル操作 |
| confirmed | completed | 完了操作 |

---

## 3. ActorType（操作者種別）

予約操作を実行したアクターの種別を表す列挙型の値オブジェクト。

| 値 | 説明 |
|----|------|
| `customer` | 顧客（LIFF アプリ経由） |
| `owner` | オーナー（Web 管理画面経由） |

使用箇所:
- Reservation.createdBy: 予約作成者の識別
- ReservationHistory.changedBy: 変更操作者の識別
- ドメインイベントの `modifiedBy`, `cancelledBy` フィールド

---

## 4. ChangeType（変更種別）

ReservationHistory に記録される変更の種別を表す列挙型の値オブジェクト。

| 値 | 説明 |
|----|------|
| `modified` | 予約日時の変更 |
| `cancelled` | 予約のキャンセル |
| `completed` | 予約の完了 |

---

## 5. ReservationDateTime（予約日時）

予約の日時を表す値オブジェクト。タイムゾーン情報を含む。

| 項目 | 内容 |
|------|------|
| 内部表現 | ISO 8601 形式のタイムゾーン付き日時 |
| タイムゾーン | JST（+09:00）固定 |
| 制約 | null 不可。過去日時での新規予約作成は不可 |
| 等価性 | タイムゾーンを正規化した上での日時一致で判定 |
| 例 | `2024-01-15T10:00:00+09:00` |

### ドメインルール

- 過去の日時を持つ予約に対する変更・キャンセル操作は禁止される
- 比較ロジック（過去日時判定）はこの値オブジェクト内に持つ

---

## 6. DurationMinutes（施術時間）

施術にかかる所要時間を分単位で表す値オブジェクト。

| 項目 | 内容 |
|------|------|
| 内部表現 | 正の整数（分） |
| 制約 | 最小 15 分、最大 1440 分（24時間）。null 不可 |
| 等価性 | 数値の一致で判定 |
| 根拠 | Unit 3 のスロット仕様（Q5）に準拠 |

備考: 予約作成時に Unit 3 のスロット情報から取得し、Reservation のスナップショット属性として保持する。

---

## 7. CustomerName（顧客名）

顧客名を表す値オブジェクト。予約作成時のスナップショットとして保持される。

| 項目 | 内容 |
|------|------|
| 内部表現 | 文字列 |
| 制約 | null 不可。空文字不可 |
| 等価性 | 文字列の一致で判定 |

備考: Unit 6 の顧客情報における `customerName` のスナップショット。イベントペイロード（Unit 5 / Unit 7 向け）にそのまま含まれる。

---

## 8. 外部コンテキスト参照値オブジェクト

以下の値オブジェクトは、他の境界づけられたコンテキスト（外部ユニット）のエンティティを参照するための識別子である。Unit 4 のドメインモデル内で値オブジェクトとして明示的に定義することで、型安全性の向上とバリデーションの集約を図る（Q20 確定）。

### 8-1. OwnerId（オーナーID）

| 項目 | 内容 |
|------|------|
| 参照先 | Unit 1（認証・アカウント管理）のオーナーエンティティ |
| 内部表現 | 文字列 |
| 制約 | null 不可。空文字不可 |
| 等価性 | 文字列の一致で判定 |

### 8-2. CustomerId（顧客ID）

| 項目 | 内容 |
|------|------|
| 参照先 | Unit 6（顧客情報管理）の顧客エンティティ |
| 内部表現 | 文字列 |
| 制約 | null 不可。空文字不可 |
| 等価性 | 文字列の一致で判定 |

### 8-3. SlotId（スロットID）

| 項目 | 内容 |
|------|------|
| 参照先 | Unit 3（スケジュール・空き枠管理）のスロットエンティティ |
| 内部表現 | 文字列 |
| 制約 | null 不可。空文字不可 |
| 等価性 | 文字列の一致で判定 |

### 8-4. LineUserId（LINE ユーザーID）

| 項目 | 内容 |
|------|------|
| 参照先 | Unit 2（LINE 連携基盤）/ LINE プラットフォームのユーザー識別子 |
| 内部表現 | 文字列（`U` + 32桁の16進数） |
| 制約 | null 許容（LINE 未連携の顧客や、LINE 未連携のオーナーの場合）。非 null の場合は `U` + 32桁hex の形式に一致すること |
| 等価性 | 文字列の一致で判定 |
| 使用箇所 | Reservation.lineUserId（顧客の LINE ID）、Reservation.ownerLineUserId（オーナーの LINE ID） |

備考: PACT の matchingRules で `^U[0-9a-f]{32}$` の正規表現が定義されている。

### 8-5. HistoryId（変更履歴ID）

| 項目 | 内容 |
|------|------|
| 内部表現 | UUID v4 形式の文字列 |
| 生成ルール | 変更履歴作成時にシステムが自動生成 |
| 制約 | null 不可 |
| 等価性 | UUID 文字列の一致で判定 |
