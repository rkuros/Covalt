# ユーザーストーリー最終版

## プロジェクト概要

**システム名**: 個人事業主向けLINE予約システム
**対象業種**: 汎用（初回ターゲット: マッサージ）
**LINE連携方式**: 方式C（Messaging API + LIFF の組み合わせ）
**インフラ構成**: AWS（CloudFront + S3 + API Gateway + Lambda）
**想定規模**: 1日数件〜数十件の予約
**実装スコープ**: 全35件（Must 16件 + Should 11件 + Could 8件）をすべて実装予定

---

## アクター定義

| アクター | 説明 |
|---|---|
| **顧客（エンドユーザー）** | LINEから予約を行う一般利用者 |
| **個人事業主（オーナー）** | 予約を管理するサービス提供者。Web管理画面を利用 |
| **システム管理者** | システム全体を管理する運用者 |

## ユーザーストーリー一覧（全35件）

### Must - MVP必須（16件）

#### EP-08: システム管理・認証

**US-S01: オーナーのWeb管理画面ログイン**
- ストーリー: オーナーとして、Web管理画面にメールアドレスとパスワードでログインしたい。それにより、自分の予約・顧客情報に安全にアクセスできる。
- アクセプタンスクライテリア:
  - [ ] メールアドレスとパスワードでログインできる
  - [ ] 誤った認証情報ではログインできず、エラーメッセージが表示される
  - [ ] ログイン後、予約一覧（ダッシュボード）が表示される
  - [ ] ログアウト操作ができる
  - [ ] 一定時間操作がない場合、セッションが自動的にタイムアウトする

**US-S03: オーナーアカウントの初期登録**
- ストーリー: システム管理者として、新規オーナーのアカウントを作成したい。それにより、オーナーがシステムを利用開始できる。
- アクセプタンスクライテリア:
  - [ ] システム管理者がオーナーのメールアドレスを登録してアカウントを作成できる
  - [ ] オーナーに初期設定用のメールが送信される
  - [ ] オーナーがメール内のリンクからパスワードを設定して利用開始できる

**US-S05: LINE公式アカウントとシステムの接続設定**
- ストーリー: システム管理者として、オーナーのLINE公式アカウントとシステムを接続設定したい。それにより、LINE経由の予約機能が動作する。
- アクセプタンスクライテリア:
  - [ ] Messaging APIのチャネルアクセストークン・チャネルシークレットを設定できる
  - [ ] LIFFアプリのIDを設定できる
  - [ ] Webhook URLが発行され、LINE Developers Consoleに設定する情報が表示される
  - [ ] 接続テスト（疎通確認）ができる

**US-S06: 個人情報の適切な取り扱い**
- ストーリー: システム管理者として、顧客の個人情報が適切に保護されるようにしたい。それにより、個人情報保護法等に準拠した運用ができる。
- アクセプタンスクライテリア:
  - [ ] 顧客の個人情報（LINEユーザーID、表示名等）は暗号化して保存される
  - [ ] API通信はすべてHTTPS経由で行われる
  - [ ] オーナーは自分の顧客情報のみアクセスでき、他のオーナーの情報にはアクセスできない
  - [ ] 顧客情報のエクスポート機能がある場合、アクセス制御が適用される

**US-S07: システムの可用性**
- ストーリー: オーナーとして、システムが安定的に稼働してほしい。それにより、予約の取りこぼしなく運用できる。
- アクセプタンスクライテリア:
  - [ ] LINE Webhookの受信が失敗した場合、リトライ処理が行われる
  - [ ] 予約の二重登録が発生しないよう、排他制御が実装されている
  - [ ] システムエラー発生時、顧客にはエラーメッセージが表示され、オーナーに通知される

#### EP-01: LINE連携基盤

**US-C01: LINE公式アカウントの友だち追加**
- ストーリー: 顧客として、LINE公式アカウントを友だち追加したい。それにより、予約システムを利用できるようになる。
- アクセプタンスクライテリア:
  - [ ] QRコードまたはリンクからLINE公式アカウントを友だち追加できる
  - [ ] 友だち追加完了時にウェルカムメッセージが自動送信される
  - [ ] ウェルカムメッセージにシステムの利用方法の簡単な案内が含まれる
  - [ ] 友だち追加時に顧客情報がシステムに自動登録される（LINEユーザーID、表示名）

**US-C02: リッチメニューからの操作**
- ストーリー: 顧客として、LINEトーク画面のリッチメニューから主要な操作にアクセスしたい。それにより、迷わずに予約や確認ができる。
- アクセプタンスクライテリア:
  - [ ] トーク画面下部にリッチメニューが常時表示される
  - [ ] リッチメニューから「予約する」をタップするとLIFFアプリ（予約画面）が起動する
  - [ ] リッチメニューから「予約確認」をタップすると予約一覧が表示される
  - [ ] リッチメニューから「予約履歴」をタップすると過去の予約履歴が表示される

**US-C03: LIFFアプリの起動と認証**
- ストーリー: 顧客として、LINEアプリ内で予約画面（LIFFアプリ）をスムーズに開きたい。それにより、別アプリへの切り替えなしに予約操作ができる。
- アクセプタンスクライテリア:
  - [ ] リッチメニューまたはメッセージ内のリンクからLIFFアプリが起動する
  - [ ] LINEの認証情報が自動的に引き継がれ、追加ログインが不要である
  - [ ] LIFFアプリ起動時にLINEユーザー情報（ユーザーID、表示名）が取得される

#### EP-04: スケジュール・空き枠管理

**US-O06: 営業時間の設定**
- ストーリー: オーナーとして、曜日ごとの営業時間を設定したい。それにより、営業時間内のみ予約を受け付けられる。
- アクセプタンスクライテリア:
  - [ ] Web管理画面から曜日ごとに営業開始時間・終了時間を設定できる
  - [ ] 曜日ごとに営業日/定休日を設定できる
  - [ ] 設定した営業時間が空きスロットの生成に反映される

**US-O08: 空きスロットの管理**
- ストーリー: オーナーとして、予約可能な空きスロット（時間枠）を作成・編集・削除したい。それにより、自分のスケジュールに合わせて予約枠を柔軟に管理できる。
- アクセプタンスクライテリア:
  - [ ] Web管理画面から特定の日付・時間帯に空きスロットを手動で作成できる
  - [ ] 既存の空きスロットを編集（時間帯の変更）できる
  - [ ] 予約が入っていない空きスロットを削除できる
  - [ ] 予約済みのスロットは削除できない（または警告が表示される）
  - [ ] スロットの所要時間（枠の長さ）を設定できる

#### EP-02: 予約管理（顧客側）

**US-C04: 空き状況の確認**
- ストーリー: 顧客として、希望する日時の空き状況を確認したい。それにより、予約可能な日時を把握した上で予約できる。
- アクセプタンスクライテリア:
  - [ ] LIFFアプリ上でカレンダーUIから日付を選択できる
  - [ ] 選択した日付の空きスロット（時間帯）が一覧表示される
  - [ ] 空きスロットは「予約可能」「予約済み」の状態が視覚的に区別できる
  - [ ] 過去の日付は選択できない
  - [ ] オーナーが設定した休業日は選択できない（または休業日であることが表示される）

**US-C05: 予約の新規作成**
- ストーリー: 顧客として、希望する日時で予約を作成したい。それにより、サービスの提供を受ける約束を確定できる。
- アクセプタンスクライテリア:
  - [ ] LIFFアプリ上で空きスロットを選択して予約を作成できる
  - [ ] 予約作成前に確認画面が表示される（日時の確認）
  - [ ] 予約作成が完了すると完了画面が表示される
  - [ ] 予約作成完了後、LINEトーク画面に予約確定メッセージが送信される
  - [ ] 同一時間帯に重複した予約はできない（二重予約防止）
  - [ ] 予約作成時にシステムが一意の予約IDを発行する

#### EP-06: 通知

**US-C10: 予約確定通知の受信**
- ストーリー: 顧客として、予約が確定したらLINEで通知を受け取りたい。それにより、予約が正しく完了したことを確認できる。
- アクセプタンスクライテリア:
  - [ ] 予約作成完了時にLINEトークに確定通知メッセージが届く
  - [ ] 通知メッセージに予約日時が記載されている
  - [ ] 通知メッセージに予約IDが記載されている

**US-O12: 新規予約通知の受信**
- ストーリー: オーナーとして、顧客が新規予約を作成した際にLINEで通知を受け取りたい。それにより、リアルタイムで予約状況を把握できる。
- アクセプタンスクライテリア:
  - [ ] 顧客が予約を作成した際、オーナーのLINEに通知が届く
  - [ ] 通知メッセージに予約日時・顧客名が記載されている

#### EP-03: 予約管理（オーナー側）

**US-O01: 予約一覧の表示**
- ストーリー: オーナーとして、Web管理画面で予約一覧を確認したい。それにより、日々の予約状況を把握できる。
- アクセプタンスクライテリア:
  - [ ] Web管理画面にログイン後、予約一覧が表示される
  - [ ] 日別・週別・月別の表示切り替えができる
  - [ ] 各予約の日時・顧客名・ステータスが一覧で確認できる
  - [ ] 予約ステータス（確定・キャンセル済み・完了）でフィルタリングできる

**US-O02: 予約の詳細確認**
- ストーリー: オーナーとして、個別の予約の詳細情報を確認したい。それにより、顧客対応を適切に行える。
- アクセプタンスクライテリア:
  - [ ] 予約一覧から個別の予約をクリックして詳細画面を開ける
  - [ ] 予約ID・日時・顧客名・ステータス・作成日時が表示される
  - [ ] 予約詳細画面から変更・キャンセル操作に遷移できる

**US-O10: 顧客詳細の確認**
- ストーリー: オーナーとして、個別の顧客の詳細情報と予約履歴を確認したい。それにより、顧客に合わせた対応ができる。
- アクセプタンスクライテリア:
  - [ ] 顧客一覧から個別の顧客をクリックして詳細画面を開ける
  - [ ] 顧客名・LINE表示名・登録日が表示される
  - [ ] その顧客の予約履歴（過去・今後）が表示される

**US-O11: 顧客情報の編集**
- ストーリー: オーナーとして、顧客の情報を編集したい。それにより、顧客情報を最新の状態に保てる。
- アクセプタンスクライテリア:
  - [ ] 顧客詳細画面から顧客名等の情報を編集できる
  - [ ] 編集内容を保存できる
  - [ ] 手動で新規顧客を登録できる（LINE連携なしの顧客にも対応）

#### EP-07: Googleカレンダー連携

**US-O14: Googleカレンダー連携の設定**
- ストーリー: オーナーとして、自分のGoogleカレンダーとシステムを連携したい。それにより、普段使っているカレンダーで予約状況を確認できる。
- アクセプタンスクライテリア:
  - [ ] Web管理画面からGoogleアカウントで認証・連携設定ができる
  - [ ] 連携するカレンダーを選択できる
  - [ ] 連携を解除できる

**US-O15: 予約のGoogleカレンダー自動同期**
- ストーリー: オーナーとして、予約が作成・変更・キャンセルされた際にGoogleカレンダーに自動反映されてほしい。それにより、手動で転記する手間が省ける。
- アクセプタンスクライテリア:
  - [ ] 予約が作成されると、Googleカレンダーに予定が自動追加される
  - [ ] 予約が変更されると、Googleカレンダーの予定が自動更新される
  - [ ] 予約がキャンセルされると、Googleカレンダーの予定が自動削除される
  - [ ] Googleカレンダーの予定に予約日時・顧客名が記載される

#### EP-08: システム管理・認証

**US-S04: オーナーアカウントの管理**
- ストーリー: システム管理者として、オーナーアカウントの一覧確認・無効化を行いたい。それにより、システム利用者を適切に管理できる。
- アクセプタンスクライテリア:
  - [ ] オーナーアカウントの一覧を確認できる
  - [ ] アカウントを有効/無効に切り替えられる
  - [ ] 無効化されたアカウントではログインできない
