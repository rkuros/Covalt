# Unit 4: 予約管理

## 境界づけられたコンテキスト

予約のライフサイクル全体を管理するコンテキスト。顧客側（LIFFアプリ）からの予約操作と、オーナー側（Web管理画面）からの予約操作の両方を含む。予約の作成・変更・キャンセル・確認・履歴をドメインとする。

## ユーザーストーリー（顧客側）

### US-C04: 空き状況の確認【Must】

**ストーリー**: 顧客として、希望する日時の空き状況を確認したい。それにより、予約可能な日時を把握した上で予約できる。

**アクセプタンスクライテリア**:
- [ ] LIFFアプリ上でカレンダーUIから日付を選択できる
- [ ] 選択した日付の空きスロット（時間帯）が一覧表示される
- [ ] 空きスロットは「予約可能」「予約済み」の状態が視覚的に区別できる
- [ ] 過去の日付は選択できない
- [ ] オーナーが設定した休業日は選択できない（または休業日であることが表示される）

---

### US-C05: 予約の新規作成【Must】

**ストーリー**: 顧客として、希望する日時で予約を作成したい。それにより、サービスの提供を受ける約束を確定できる。

**アクセプタンスクライテリア**:
- [ ] LIFFアプリ上で空きスロットを選択して予約を作成できる
- [ ] 予約作成前に確認画面が表示される（日時の確認）
- [ ] 予約作成が完了すると完了画面が表示される
- [ ] 予約作成完了後、LINEトーク画面に予約確定メッセージが送信される
- [ ] 同一時間帯に重複した予約はできない（二重予約防止）
- [ ] 予約作成時にシステムが一意の予約IDを発行する

---

### US-C06: 予約の変更【Should】

**ストーリー**: 顧客として、作成済みの予約の日時を変更したい。それにより、予定が変わった場合にキャンセルせずに対応できる。

**アクセプタンスクライテリア**:
- [ ] 自分の予約一覧から変更したい予約を選択できる
- [ ] 変更先の空きスロットを選択できる
- [ ] 変更前に確認画面が表示される（変更前後の日時の比較）
- [ ] 変更が完了すると、LINEトーク画面に変更完了メッセージが送信される
- [ ] 変更先の時間帯が既に予約済みの場合は変更できない
- [ ] 過去の予約は変更できない

---

### US-C07: 予約のキャンセル【Should】

**ストーリー**: 顧客として、作成済みの予約をキャンセルしたい。それにより、行けなくなった場合に事前に取り消しができる。

**アクセプタンスクライテリア**:
- [ ] 自分の予約一覧からキャンセルしたい予約を選択できる
- [ ] キャンセル前に確認画面が表示される
- [ ] キャンセルが完了すると、LINEトーク画面にキャンセル完了メッセージが送信される
- [ ] キャンセルされたスロットは再び「予約可能」状態に戻る
- [ ] 過去の予約はキャンセルできない

---

### US-C08: 予約一覧の確認【Should】

**ストーリー**: 顧客として、自分の今後の予約一覧を確認したい。それにより、予約状況をいつでも把握できる。

**アクセプタンスクライテリア**:
- [ ] リッチメニューまたはLIFFアプリから今後の予約一覧を表示できる
- [ ] 各予約の日時・ステータス（確定・キャンセル済み等）が表示される
- [ ] 予約一覧から個別の予約の詳細を確認できる
- [ ] 予約一覧から変更・キャンセル操作に遷移できる

---

### US-C09: 予約履歴の確認【Could】

**ストーリー**: 顧客として、過去の予約履歴を確認したい。それにより、これまでの利用状況を振り返ることができる。

**アクセプタンスクライテリア**:
- [ ] 過去の予約（完了済み・キャンセル済み）を一覧で確認できる
- [ ] 各予約の日時・ステータスが表示される
- [ ] 直近の履歴から順に表示される

---

## ユーザーストーリー（オーナー側）

### US-O01: 予約一覧の表示【Must】

**ストーリー**: オーナーとして、Web管理画面で予約一覧を確認したい。それにより、日々の予約状況を把握できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面にログイン後、予約一覧が表示される
- [ ] 日別・週別・月別の表示切り替えができる
- [ ] 各予約の日時・顧客名・ステータスが一覧で確認できる
- [ ] 予約ステータス（確定・キャンセル済み・完了）でフィルタリングできる

---

### US-O02: 予約の詳細確認【Must】

**ストーリー**: オーナーとして、個別の予約の詳細情報を確認したい。それにより、顧客対応を適切に行える。

**アクセプタンスクライテリア**:
- [ ] 予約一覧から個別の予約をクリックして詳細画面を開ける
- [ ] 予約ID・日時・顧客名・ステータス・作成日時が表示される
- [ ] 予約詳細画面から変更・キャンセル操作に遷移できる

---

### US-O03: 予約の手動作成【Should】

**ストーリー**: オーナーとして、電話等で受けた予約をWeb管理画面から手動で登録したい。それにより、LINE以外からの予約もシステムで一元管理できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面から日時を指定して予約を手動作成できる
- [ ] 既存の顧客を選択、または新規顧客情報を入力して予約を作成できる
- [ ] 空きスロットのみ選択可能で、二重予約が防止される
- [ ] 手動作成した予約もシステムの予約一覧に反映される

---

### US-O04: 予約の変更（オーナー操作）【Should】

**ストーリー**: オーナーとして、Web管理画面から予約の日時を変更したい。それにより、顧客からの依頼や都合に応じて柔軟に対応できる。

**アクセプタンスクライテリア**:
- [ ] 予約詳細画面から日時の変更ができる
- [ ] 変更先の空きスロットのみ選択可能である
- [ ] 変更完了後、該当顧客にLINEで変更通知が送信される（LINE連携済みの顧客の場合）
- [ ] 変更前後の日時が変更履歴として記録される

---

### US-O05: 予約のキャンセル（オーナー操作）【Should】

**ストーリー**: オーナーとして、Web管理画面から予約をキャンセルしたい。それにより、急な都合変更にも対応できる。

**アクセプタンスクライテリア**:
- [ ] 予約詳細画面からキャンセル操作ができる
- [ ] キャンセル前に確認ダイアログが表示される
- [ ] キャンセル完了後、該当顧客にLINEでキャンセル通知が送信される（LINE連携済みの顧客の場合）
- [ ] キャンセルされたスロットは再び「予約可能」状態に戻る
