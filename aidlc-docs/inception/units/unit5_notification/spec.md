# Unit 5: 通知

## 境界づけられたコンテキスト

予約イベントに応じたLINE通知の送信を管理するコンテキスト。Messaging APIを通じて、顧客向け・オーナー向けの各種通知（確定・変更・キャンセル・リマインダー）を担う。

## ユーザーストーリー（顧客向け）

### US-C10: 予約確定通知の受信【Must】

**ストーリー**: 顧客として、予約が確定したらLINEで通知を受け取りたい。それにより、予約が正しく完了したことを確認できる。

**アクセプタンスクライテリア**:
- [ ] 予約作成完了時にLINEトークに確定通知メッセージが届く
- [ ] 通知メッセージに予約日時が記載されている
- [ ] 通知メッセージに予約IDが記載されている

---

### US-C11: 予約変更通知の受信【Should】

**ストーリー**: 顧客として、予約が変更されたらLINEで通知を受け取りたい。それにより、変更内容を確認できる。

**アクセプタンスクライテリア**:
- [ ] 予約変更完了時にLINEトークに変更通知メッセージが届く
- [ ] 通知メッセージに変更前後の日時が記載されている
- [ ] オーナー側から予約が変更された場合にも通知が届く

---

### US-C12: 予約キャンセル通知の受信【Should】

**ストーリー**: 顧客として、予約がキャンセルされたらLINEで通知を受け取りたい。それにより、キャンセルが正しく処理されたことを確認できる。

**アクセプタンスクライテリア**:
- [ ] 予約キャンセル完了時にLINEトークにキャンセル通知メッセージが届く
- [ ] 通知メッセージにキャンセルされた予約の日時が記載されている
- [ ] オーナー側から予約がキャンセルされた場合にも通知が届く

---

### US-C13: 予約リマインダー通知の受信【Could】

**ストーリー**: 顧客として、予約日の前にリマインダー通知を受け取りたい。それにより、予約を忘れずに来店できる。

**アクセプタンスクライテリア**:
- [ ] 予約日の前日にLINEトークにリマインダーメッセージが届く
- [ ] リマインダーメッセージに予約日時が記載されている
- [ ] キャンセル済みの予約にはリマインダーが送信されない

---

## ユーザーストーリー（オーナー向け）

### US-O12: 新規予約通知の受信【Must】

**ストーリー**: オーナーとして、顧客が新規予約を作成した際にLINEで通知を受け取りたい。それにより、リアルタイムで予約状況を把握できる。

**アクセプタンスクライテリア**:
- [ ] 顧客が予約を作成した際、オーナーのLINEに通知が届く
- [ ] 通知メッセージに予約日時・顧客名が記載されている

---

### US-O13: 予約変更・キャンセル通知の受信【Should】

**ストーリー**: オーナーとして、顧客が予約を変更またはキャンセルした際にLINEで通知を受け取りたい。それにより、スケジュールの変更を即座に把握できる。

**アクセプタンスクライテリア**:
- [ ] 顧客が予約を変更した際、オーナーのLINEに変更通知が届く
- [ ] 顧客が予約をキャンセルした際、オーナーのLINEにキャンセル通知が届く
- [ ] 通知メッセージに予約日時・顧客名・変更内容が記載されている
