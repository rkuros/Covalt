# Unit 1: 認証・アカウント管理

## 境界づけられたコンテキスト

オーナーおよびシステム管理者の認証・アカウントライフサイクルを管理するコンテキスト。Web管理画面へのアクセス制御、アカウントの作成・無効化を担う。

## ユーザーストーリー

### US-S01: オーナーのWeb管理画面ログイン【Must】

**ストーリー**: オーナーとして、Web管理画面にメールアドレスとパスワードでログインしたい。それにより、自分の予約・顧客情報に安全にアクセスできる。

**アクセプタンスクライテリア**:
- [ ] メールアドレスとパスワードでログインできる
- [ ] 誤った認証情報ではログインできず、エラーメッセージが表示される
- [ ] ログイン後、予約一覧（ダッシュボード）が表示される
- [ ] ログアウト操作ができる
- [ ] 一定時間操作がない場合、セッションが自動的にタイムアウトする

---

### US-S02: オーナーのパスワードリセット【Should】

**ストーリー**: オーナーとして、パスワードを忘れた場合にリセットしたい。それにより、アカウントへのアクセスを回復できる。

**アクセプタンスクライテリア**:
- [ ] ログイン画面から「パスワードを忘れた方」リンクにアクセスできる
- [ ] 登録メールアドレスにパスワードリセット用リンクが送信される
- [ ] リセットリンクから新しいパスワードを設定できる
- [ ] リセットリンクには有効期限がある

---

### US-S03: オーナーアカウントの初期登録【Must】

**ストーリー**: システム管理者として、新規オーナーのアカウントを作成したい。それにより、オーナーがシステムを利用開始できる。

**アクセプタンスクライテリア**:
- [ ] システム管理者がオーナーのメールアドレスを登録してアカウントを作成できる
- [ ] オーナーに初期設定用のメールが送信される
- [ ] オーナーがメール内のリンクからパスワードを設定して利用開始できる

---

### US-S04: オーナーアカウントの管理【Could】

**ストーリー**: システム管理者として、オーナーアカウントの一覧確認・無効化を行いたい。それにより、システム利用者を適切に管理できる。

**アクセプタンスクライテリア**:
- [ ] オーナーアカウントの一覧を確認できる
- [ ] アカウントを有効/無効に切り替えられる
- [ ] 無効化されたアカウントではログインできない
