- [x] **ステップ4: LINE連携の技術要件の明確化**
  - LINE側の連携方式を明確にする

  [Question] LINE連携について、以下のどの方式を想定していますか？
  - **A**: LINE公式アカウント + Messaging API（チャットボット形式で予約を進める）
  - **B**: LIFF（LINE Front-end Framework）アプリ（LINE内でWebアプリとして動作する予約画面）
  - **C**: AとBの組み合わせ
  - **D**: その他（具体的に教えてください）
  - **E**: 技術選定はまだ決めていない（ユーザーストーリーでは技術に依存しない形で記述する）

  [Answer]　Eですが、AとBのプロスコンスがわかりません。それらを詳しく調査してもらって、ABCのどの手段で行くか決めたいです。参考になるのはhttps://linepi.humanoid-inc.net/privacyらしいです。

  **調査結果**: [line_integration_comparison.md](./line_integration_comparison.md) を参照

  [Question] 調査結果を踏まえ、どの方式を採用しますか？
  - **A**: Messaging API（チャットボット形式） - 推奨。低コスト・開発が簡潔・将来Cへ移行可能
  - **B**: LIFF（Webアプリ形式） - UI自由度は高いが開発コスト増。通知にはMessaging APIも必要
  - **C**: A + B の組み合わせ - 最もリッチだが開発・運用コスト最大

  [Answer] C（Messaging API + LIFF の組み合わせ）。LIFFのバックエンドはAWS（CloudFront + S3 + API Gateway + Lambda）構成。
