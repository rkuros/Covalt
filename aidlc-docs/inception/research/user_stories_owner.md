# ユーザーストーリー: 個人事業主（オーナー）

## EP-03: 予約管理（オーナー側）

### US-O01: 予約一覧の表示

**ストーリー**: オーナーとして、Web管理画面で予約一覧を確認したい。それにより、日々の予約状況を把握できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面にログイン後、予約一覧が表示される
- [ ] 日別・週別・月別の表示切り替えができる
- [ ] 各予約の日時・顧客名・ステータスが一覧で確認できる
- [ ] 予約ステータス（確定・キャンセル済み・完了）でフィルタリングできる

### US-O02: 予約の詳細確認

**ストーリー**: オーナーとして、個別の予約の詳細情報を確認したい。それにより、顧客対応を適切に行える。

**アクセプタンスクライテリア**:
- [ ] 予約一覧から個別の予約をクリックして詳細画面を開ける
- [ ] 予約ID・日時・顧客名・ステータス・作成日時が表示される
- [ ] 予約詳細画面から変更・キャンセル操作に遷移できる

### US-O03: 予約の手動作成

**ストーリー**: オーナーとして、電話等で受けた予約をWeb管理画面から手動で登録したい。それにより、LINE以外からの予約もシステムで一元管理できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面から日時を指定して予約を手動作成できる
- [ ] 既存の顧客を選択、または新規顧客情報を入力して予約を作成できる
- [ ] 空きスロットのみ選択可能で、二重予約が防止される
- [ ] 手動作成した予約もシステムの予約一覧に反映される

### US-O04: 予約の変更（オーナー操作）

**ストーリー**: オーナーとして、Web管理画面から予約の日時を変更したい。それにより、顧客からの依頼や都合に応じて柔軟に対応できる。

**アクセプタンスクライテリア**:
- [ ] 予約詳細画面から日時の変更ができる
- [ ] 変更先の空きスロットのみ選択可能である
- [ ] 変更完了後、該当顧客にLINEで変更通知が送信される（LINE連携済みの顧客の場合）
- [ ] 変更前後の日時が変更履歴として記録される

### US-O05: 予約のキャンセル（オーナー操作）

**ストーリー**: オーナーとして、Web管理画面から予約をキャンセルしたい。それにより、急な都合変更にも対応できる。

**アクセプタンスクライテリア**:
- [ ] 予約詳細画面からキャンセル操作ができる
- [ ] キャンセル前に確認ダイアログが表示される
- [ ] キャンセル完了後、該当顧客にLINEでキャンセル通知が送信される（LINE連携済みの顧客の場合）
- [ ] キャンセルされたスロットは再び「予約可能」状態に戻る

## EP-04: スケジュール・空き枠管理

### US-O06: 営業時間の設定

**ストーリー**: オーナーとして、曜日ごとの営業時間を設定したい。それにより、営業時間内のみ予約を受け付けられる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面から曜日ごとに営業開始時間・終了時間を設定できる
- [ ] 曜日ごとに営業日/定休日を設定できる
- [ ] 設定した営業時間が空きスロットの生成に反映される

### US-O07: 休業日の設定

**ストーリー**: オーナーとして、特定の日付を休業日として設定したい。それにより、臨時休業や祝日等に予約が入らないようにできる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面からカレンダー上で特定の日付を休業日に設定できる
- [ ] 休業日に設定された日は顧客側の空き状況に表示されない
- [ ] 既に予約が入っている日を休業日に設定しようとした場合、警告が表示される
- [ ] 休業日の設定を解除できる

### US-O08: 空きスロットの管理

**ストーリー**: オーナーとして、予約可能な空きスロット（時間枠）を作成・編集・削除したい。それにより、自分のスケジュールに合わせて予約枠を柔軟に管理できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面から特定の日付・時間帯に空きスロットを手動で作成できる
- [ ] 既存の空きスロットを編集（時間帯の変更）できる
- [ ] 予約が入っていない空きスロットを削除できる
- [ ] 予約済みのスロットは削除できない（または警告が表示される）
- [ ] スロットの所要時間（枠の長さ）を設定できる

## EP-05: 顧客情報管理

### US-O09: 顧客一覧の表示

**ストーリー**: オーナーとして、顧客の一覧を確認したい。それにより、全体の顧客状況を把握できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面で顧客一覧が表示される
- [ ] 顧客名で検索できる
- [ ] LINE連携済みかどうかが表示される

### US-O10: 顧客詳細の確認

**ストーリー**: オーナーとして、個別の顧客の詳細情報と予約履歴を確認したい。それにより、顧客に合わせた対応ができる。

**アクセプタンスクライテリア**:
- [ ] 顧客一覧から個別の顧客をクリックして詳細画面を開ける
- [ ] 顧客名・LINE表示名・登録日が表示される
- [ ] その顧客の予約履歴（過去・今後）が表示される

### US-O11: 顧客情報の編集

**ストーリー**: オーナーとして、顧客の情報を編集したい。それにより、顧客情報を最新の状態に保てる。

**アクセプタンスクライテリア**:
- [ ] 顧客詳細画面から顧客名等の情報を編集できる
- [ ] 編集内容を保存できる
- [ ] 手動で新規顧客を登録できる（LINE連携なしの顧客にも対応）

## EP-06: 通知（オーナー向け）

### US-O12: 新規予約通知の受信

**ストーリー**: オーナーとして、顧客が新規予約を作成した際にLINEで通知を受け取りたい。それにより、リアルタイムで予約状況を把握できる。

**アクセプタンスクライテリア**:
- [ ] 顧客が予約を作成した際、オーナーのLINEに通知が届く
- [ ] 通知メッセージに予約日時・顧客名が記載されている

### US-O13: 予約変更・キャンセル通知の受信

**ストーリー**: オーナーとして、顧客が予約を変更またはキャンセルした際にLINEで通知を受け取りたい。それにより、スケジュールの変更を即座に把握できる。

**アクセプタンスクライテリア**:
- [ ] 顧客が予約を変更した際、オーナーのLINEに変更通知が届く
- [ ] 顧客が予約をキャンセルした際、オーナーのLINEにキャンセル通知が届く
- [ ] 通知メッセージに予約日時・顧客名・変更内容が記載されている

## EP-07: Googleカレンダー連携

### US-O14: Googleカレンダー連携の設定

**ストーリー**: オーナーとして、自分のGoogleカレンダーとシステムを連携したい。それにより、普段使っているカレンダーで予約状況を確認できる。

**アクセプタンスクライテリア**:
- [ ] Web管理画面からGoogleアカウントで認証・連携設定ができる
- [ ] 連携するカレンダーを選択できる
- [ ] 連携を解除できる

### US-O15: 予約のGoogleカレンダー自動同期

**ストーリー**: オーナーとして、予約が作成・変更・キャンセルされた際にGoogleカレンダーに自動反映されてほしい。それにより、手動で転記する手間が省ける。

**アクセプタンスクライテリア**:
- [ ] 予約が作成されると、Googleカレンダーに予定が自動追加される
- [ ] 予約が変更されると、Googleカレンダーの予定が自動更新される
- [ ] 予約がキャンセルされると、Googleカレンダーの予定が自動削除される
- [ ] Googleカレンダーの予定に予約日時・顧客名が記載される
