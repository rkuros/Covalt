# ユーザーストーリー: システム・共通

## EP-08: システム管理・認証

### US-S01: オーナーのWeb管理画面ログイン

**ストーリー**: オーナーとして、Web管理画面にメールアドレスとパスワードでログインしたい。それにより、自分の予約・顧客情報に安全にアクセスできる。

**アクセプタンスクライテリア**:
- [ ] メールアドレスとパスワードでログインできる
- [ ] 誤った認証情報ではログインできず、エラーメッセージが表示される
- [ ] ログイン後、予約一覧（ダッシュボード）が表示される
- [ ] ログアウト操作ができる
- [ ] 一定時間操作がない場合、セッションが自動的にタイムアウトする

### US-S02: オーナーのパスワードリセット

**ストーリー**: オーナーとして、パスワードを忘れた場合にリセットしたい。それにより、アカウントへのアクセスを回復できる。

**アクセプタンスクライテリア**:
- [ ] ログイン画面から「パスワードを忘れた方」リンクにアクセスできる
- [ ] 登録メールアドレスにパスワードリセット用リンクが送信される
- [ ] リセットリンクから新しいパスワードを設定できる
- [ ] リセットリンクには有効期限がある

### US-S03: オーナーアカウントの初期登録

**ストーリー**: システム管理者として、新規オーナーのアカウントを作成したい。それにより、オーナーがシステムを利用開始できる。

**アクセプタンスクライテリア**:
- [ ] システム管理者がオーナーのメールアドレスを登録してアカウントを作成できる
- [ ] オーナーに初期設定用のメールが送信される
- [ ] オーナーがメール内のリンクからパスワードを設定して利用開始できる

### US-S04: オーナーアカウントの管理

**ストーリー**: システム管理者として、オーナーアカウントの一覧確認・無効化を行いたい。それにより、システム利用者を適切に管理できる。

**アクセプタンスクライテリア**:
- [ ] オーナーアカウントの一覧を確認できる
- [ ] アカウントを有効/無効に切り替えられる
- [ ] 無効化されたアカウントではログインできない

### US-S05: LINE公式アカウントとシステムの接続設定

**ストーリー**: システム管理者として、オーナーのLINE公式アカウントとシステムを接続設定したい。それにより、LINE経由の予約機能が動作する。

**アクセプタンスクライテリア**:
- [ ] Messaging APIのチャネルアクセストークン・チャネルシークレットを設定できる
- [ ] LIFFアプリのIDを設定できる
- [ ] Webhook URLが発行され、LINE Developers Consoleに設定する情報が表示される
- [ ] 接続テスト（疎通確認）ができる

## 共通: セキュリティ・個人情報保護

### US-S06: 個人情報の適切な取り扱い

**ストーリー**: システム管理者として、顧客の個人情報が適切に保護されるようにしたい。それにより、個人情報保護法等に準拠した運用ができる。

**アクセプタンスクライテリア**:
- [ ] 顧客の個人情報（LINEユーザーID、表示名等）は暗号化して保存される
- [ ] API通信はすべてHTTPS経由で行われる
- [ ] オーナーは自分の顧客情報のみアクセスでき、他のオーナーの情報にはアクセスできない
- [ ] 顧客情報のエクスポート機能がある場合、アクセス制御が適用される

### US-S07: システムの可用性

**ストーリー**: オーナーとして、システムが安定的に稼働してほしい。それにより、予約の取りこぼしなく運用できる。

**アクセプタンスクライテリア**:
- [ ] LINE Webhookの受信が失敗した場合、リトライ処理が行われる
- [ ] 予約の二重登録が発生しないよう、排他制御が実装されている
- [ ] システムエラー発生時、顧客にはエラーメッセージが表示され、オーナーに通知される
